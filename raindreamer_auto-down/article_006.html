Hackerrank Week 9 Solution

2th of September 2014

Contest Link

I had attended this contest recently, and it turns out that most of the problems are very talented, though some are a little stereotyped.

Sansa and XOR

Calculate XOR value of XOR of all sub-array.

XOR is commutative, so once we know the occurence number of an element, we can simply XOR all elements which occur for odd times.

Lexicographic paths

To get (X, Y) only by horizontal or vertical moves, find the k-th lexicographic small path. (X, Y <= 10)

By combination, we can know the way to get (X, Y) for (0, 0) is C(X + Y, Y), so enumerate the first move is H or V, and check the left number of ways is larger than K.

Mixing proteins

Given a mutation matrix: 

    A   B   C   D
    _   _   _   _
A|  A   B   C   D
B|  B   A   D   C
C|  C   D   A   B
D|  D   C   B   A

Now given a string(len<=10^6), for a time unit, each character in string will change in the way the matrix describe, with the next character, i.e. S(i, T + 1) = M(S(i, T), S(i + 1, T)). Find the string after K(<=10^9) time unit.

Observe this matrix, we can easily find that M(i, j) = M(j, i). And also M(M(i, k), M(j, k)) = M(i, j). So we can deduce that: 

S(i, T + 2) 
= M( M(S(i, T), S(i, T + 1)), M(S(i, T + 1), S(i, T + 2)) )
= M( M(S(i, T), S(i, T + 1)), M(S(i, T + 2), S(i, T + 1)) )
= M( S(i, T), S(i, T + 2) )

It's a beautiful formula! So S(i, T + k) = M( S(i, T), S(i, T + k) )? Unfortunately, it's not ture. When k = 3, we can't simplify the formula in this way.

But for K = 2^T(T=0, 1, 2...), the formula share common k in the second step, so it does work. Then we can calculate the final answer in O(len * log(K)) time.

A Game of Knights

Two player move N knights (0 <= Xi, Yi < 300) in a checkboard alternately. In each move, the player can choose at most K knights and move each knight at most X move singly. Determine the winner of the game.

First we need get the SG value for each grid, as X <= 5, we can simple dfs all possible succeed state, the time complexity is about O(300 * 300 * 2 ^ 5), kind of slow. As usual, my method is printing the value table, and find the pattern of SG value by naked eye.

And it do have a pattern: SG[x][y] = SG[x - 2 * X - 1][y - 2 * X - 1], and for x < 2*X+1 and y >= 4 * X, sg[x][y] = x. With this two optimizations, we can get the SG vaule of the checkboard grids very fast. (But I haven't proved why the pattern works)

The left problem is a standrad K-Nim game, we can convert the sg value to binary, and do a (K+1)-nary non-carried addition for all SG vaule, just as the simple Nim game.

Alex vs Fedor

Calculate the number of Minimal Spanning Tree and Spanning Tree for a graph.

This is an extremely speical and traditional algorithm to solve the problem: Kirchhoff's Matrix .

My solution failed in some final testcases for int64 problem, though the original number in Kirchhoffs' matrix must fit in int, but while calculating the value of determinant, the value may be overflowed.
