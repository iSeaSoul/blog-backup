C++技巧系列03 - 利用define

26th of January 2015

cplusplus trick learning-note

在很多规范中， define 的使用是不被鼓励的，可能很多太过随意的使用造成了大量的问题。但任何事物存在都是有价值的，正确的使用 define 可以帮助清晰代码、减少冗余。 

#define make_name(name) _##name##_name
#define print_name(x) cout << #x << ' ' << x << endl;

int main() {
    int make_name(li) = 1;
    print_name(_li_name);

    return 0;
}

这段代码演示了 # 与 ## 的功能：前者表示宏传入的内容，后者用这个内容内嵌到字符串中。

Yes, find that? 如果这里有一大串重复且规则相同的名字（因为变量名都是自己取的，这种情况非常常见），这个宏是不是很有意义？

如果要对着一大串相同规则的名字做一些操作，是不是可以抽象出一些范式？

计算机的核心就在于数据抽象和逻辑抽象，永远不要忘了。
