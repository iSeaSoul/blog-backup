A. Design Tutorial: Learn from Math

Divide a integer into two composite numbers.

As N <= 10^6, we can simply generate all primes and iterate them. Actually, for a even number, divide it into 4 and n - 4, for a odd number, divide it into 9 and n - 9.

B. Design Tutorial: Learn from Life

Transport N people to the corresponding floors and then return the elevator to the first floor, with a constraint that K people for each turn.

Sort the floors the people want to move, and transport them greedily: the maximum K people in the left group till nobody was left.

C. Design Tutorial: Make It Nondeterministic

There are N people, each person will use either his/her first name or last name as a handle. Can the lexicographical order of the handles be exactly equal to the given permutation p?

Always choose the lexicographical smallest name while iterate the given permutation. If no possible name can be chosen, impossible.

D. Design Tutorial: Inverse the Problem

Given an n×n distance matrix. Determine if it is the distance matrix of a weighted tree (all weights must be positive integers).

During the contest, I use a strange method: try to generate the tree recursively, for a state build_tree(x, f) , find edge (V, dis) with the minimum length in which dis[x] is less than dis[f], add this edge to the tree. Formally, it's a variety of building the MST : )

F. Design Tutorial: Change the Goal

Given n integers x1,x2,...,xn and n integers y1,y2,...,yn as goal, use the operation xi ^= xj to make them equal.

I haven't solved it during the contest, so I learned from the solutions and editorial after the ending of the contest. The basic idea is simple and brilliant.

Generate the vector space of Xi and Yi first, by Gauss elimination. As Gauss elimination only needs two kinds of operations:

  • Swap two numbers Xi and Xj: Xi ^= Xj ^= Xi ^= Xj
  • XOR two numbers Xi and Xj: Xi ^= Xj

And this operation is inversible. So we can only use operation xi ^= xj change them to two base (only with rank rows) matrix. Now the left task is checking whether the first base MX can change to the second base MY. For any key column in the base matrix MX, if there is a different state between MXi and MYi in this bit, we need XOR the MXi with MX_withthisbit. Then check the identity of the two base matrix.
