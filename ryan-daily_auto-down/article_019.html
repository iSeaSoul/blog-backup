Linux Shell Features

26th of February 2015

linux learning-note

1.Wildcards & Brace expansions

Linux的命令都可以支持通配符匹配。比如这两个命令： 

$ ls a*
$ ls [^a]*

就等价于 

$ ls a apple ability
$ ls b beauty curiosity

也就是将文件夹类的所有文件匹配这个模式。支持 

*
?
[set]
[^set]

Linux命令也支持括号表达式。 

$ echo {a,b}{a,b}
aa ab ba bb
$ ls {test_,ex_}[ab]*
test_a.h ex_b.cpp

2.Shell variables 

$ TEST=2
$ echo $TEST
2

基本的使用方法是这样的。有一些是系统环境变量，可以通过 

$ printenv

查看。如果需要修改环境变量，比如加入到 PATH 中 

$ export PATH=~/prog/bin:$PATH

3.Aliases

命令别名可以使用 alias 实现： 

$ alias ll='ls -l'

将这些缩写添加到 .bashrc 中，就可以直接使用了。

4.Input/output redirection & Pipes & Combining commands

输入输出流使用 < 和 > 重定向。 

$ prog < in > out 2> error
$ prog >& out

第二个命令表示error信息也输出到out中。

管道就是将前一个的输出当做后一个命令的输入。 

$ who | sort | awk '{print $1}'

使用 ; 连接命令就是顺序执行，使用 & 一个失败就停止，使用 | 一个成功就停止。这和高级语言的语义是一致的。

5.Quoting & Escaping

Linux有三种引用： 

$ echo 'The variable HOME has value $HOME'
The variable HOME has value $HOME
$ echo "The variable HOME has value $HOME"
The variable HOME has value /home/smith
$ echo My name is `whoami`
My name is smith

简单来说，第一种不转义，第二中替换变量内容，第三种内容是shell命令的输出。

如果前面就一个 \ ，则字符不转义。包括前面说的Wildcards等等。

6.Command history

命令历史的常用方法为：

  • history print your history
  • !! Re-run previous command
  • !comm Re-run previous command begin with comm
  • !N Re-run N-th command in history
  • !-N Re-run current-N-th command in history
  • !$ last parameter of the last command
  • !* all parameters of the last command
