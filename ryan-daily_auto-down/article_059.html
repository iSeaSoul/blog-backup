1.1 程序设计的基本元素（Lisp/Scheme语言简介）

Lisp语言的核心是 过程->数据 。

  • 前缀表达式： (+ 2 3 4)
  • define定义变量（variable）和过程（method）
  • 嵌套的过程定义：`expression ::= (method name expression 1... expression_n) | variable
  • 解释器用值替换表达式中的形式参数来完成过程调用
  • 正则序：先完全展开后求值 VS 应用序：先求值再展开（Lisp用后者）
  • (cond (<pi> <ei>) (else <ej>))
  • if <predicate> <consequent> <alternative>
  • 过程的形式参数是约束变量，是过程内的局部名字

1.5 对于程序 

(define (p) (p))

(define (test x y) (if (= x 0) 0 y))

执行 (test 0 (p)) 。正则序和应用序有什么区别？

正则序会先展开： (if (= x 0) 0 (p)) 而没有求值，if判断成功，返回0。 而应用序会先尝试求值，(p)会直接报错，或者没有结果。（DrRacket里面没有执行结果）

1.6 如果定义 

(define (new-if predicate then-clause else-clause)
  (cond (predicate then-clause)
     (else else-clause)))

new-if和if的区别在哪里？

if是短路的；而这里的new-if是需要展开的。如果是应用序（Lisp/Scheme的确是），可能有一些不受控制的死循环产生。

1.7 迭代法求sqrt直到新的guess值差距很小时停止。 

(define (sqrt x)
  (sqrt-iter 1.0 x))

(define (sqrt-iter guess x)
  (if (good_enough? (improve guess x) guess) 
      guess
      (sqrt-iter (improve guess x) x)))

(define (improve guess x)
  (average guess (/ x guess)))

(define (average x y)
  (/ (+ x y) 2))

(define (good_enough? new_guess guess)
  (< (/ (abs (- new_guess guess)) guess) 0.0000001))
