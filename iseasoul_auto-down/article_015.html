Interviewstreet Math Problems

30th of July 2012

Recent afternoon time are almost wasted at there, finally with all math problems done.

So, write some simple solutions to record it.

EQUATION

For an equation 1/x + 1/y = 1/num, in which num is known. The solution satisfies the condition x = num + c, y = num + (num^2) / c. You can verify this by substituting this condition.

Grid Walking

First, use dynamic programming to calculate the ways of walk X steps without exceeding the lower and upper bound in each dimension. Then use another DP and combination to calculate the total ways when distributing the M steps to N dimensions.

Circle Summation

Typical Matrix Multiplication. If shiftting all numbers to its left place circularly, then the operation of the second line is totally the same with the first line, do this similarily in the following lines.

Stone piles

Game Theory with SG function. Use a depth-first-search to enumerate all possible circumstances after a divide operation, calculate all SG values and XOR them.

Points in a Plane

With bitmask, get all state where all point(s) are on a line first. Then enumerate all valid states to remove, calculate the minimum steps to achieve current state. Enumeration here can use the clever 3^N subsets trick to handle. But it's still exceeding the time limit for the last test case. Need an interesting improvement to avoid some duplicated calculation.

String Reduction

A plenty of greedy algorithm works.

Permutation game

Simple bitset DP.

Connect the country

A similar problem, connect totally two random cities whether connected or not, here only connect two random cities which is not connected. Think DP for a long time, but it asks to record the existed edge numbers in each connected component, too heavy for implement. While the vertex number is only 30 and just asks for integer part, simulation is enough, yes, try millions times of experiments.

Unfriendly Numbers

K has at most 10000 factors, problem solved.

Ants

A fact: if distance between two ants is 1, they can meet 200001 times, otherwises 200000 times. So the only task is processing the adjacent ants.
