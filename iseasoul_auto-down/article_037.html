Some Common Mistakes in ASP.NET

16th of April 2012

Last time, about half a month ago, I fixed it one by one with Google.

Today, I fixed it one by one again, with Google, still. Read, translate, try. Doesn't work, read, translate, try. WTF.

But it should be memo to prevent me forgetting again and again.

1) XXX can't find or somehow, the default version of "Application Pools"is v2.0, so change this to v4.0 first.

2) Login failed for user 'XXX\XXX'. Change the identity of the pool to "LocalSystem". Or add a new accout in DB, and grant it.

3) If change to"LocalSystem", the website will run under "NT AUTHORITY\SYSTEM", if still not working, add this accout to your website's DB and try again.

4) Service Unavailable. HTTP Error 503. Restart the IIS, if failed, delete it, add the port back and do the above again.
