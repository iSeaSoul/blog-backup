275 - 求一个N的字典序最小排列，逆序数要大于S，且字典序大于给出的一个排列P。N <= 20。P可能长度小于N。

先补齐P，后面按没有出现的字母（数字）升序排列。然后从高位到低位，对每一位贪心求出大于P排列且逆序对在最好情况（后面的全部逆序）时能大于S的最小数字。

500 - 很绕口的一个题。大意就是在一个 H * L 的格子里选出K个点，这K个点可以用一条从x轴出发且 0 <= xi <= L 的射线连接起来。求选取方案 % mod。范围都为 [1, 2000]。

考虑直线穿过的单元格，步长为 (dx, dy)，枚举这个步长后再枚举起点就OK了。但复杂度N^3，需要优化一层。考虑枚举起点这一层，这样得到的可选点数P_select是 min(H / dy, (L - x0) / dx), 在这些点中选出K个。那么可以发现备选点数P_select_x从大到小的顺序是 dx个0， dx个1， dx个2... dx个L/dx，剩余的若干个。然后与H/dy取一个较小值。那么枚举就没有意义了，只需要求一个部分和。注意一下边界情况。

975 - 将N个数分为两组，两组的AND值相等的方案数。N <= 50，num_in_array <= 2^20。
