Floyd：DP在图论中（一）

2th of March 2010

默认分类

计算所有点的最近点对，floyd是一个很有效也很简洁的算法。
三重循环，更新操作，一个标准O(V^3)算法，也许会有很多质疑，用堆优化的dijkatra可以达到O(VlogV)，那么做V次dijkstra也只是O(V*V*logV)，但是有一点无法被忽视，五行代码，还可以处理带负权的边，优势毋庸再说。

Q1 Dij(k)的含义是什么？
如果把初值Dij(0)置为g[i][j]，那么Dij(k)表示仅通过前1~k个顶点得到的最短路。

Q2 如何证明Dij(k) = min{Dij(k-1), Dik(k-1) + Dkj(k-1)}的正确性？
设Dij(k)对应的最短路为i v1 e1 …ep-1 vp j，若k不属于其中的顶点，那么这条最短路也对应Dij(k-1)中的最短路，若k属于不属于其中的顶点，那么可以分为i v1 e1 …vs-1 vs k 和 k vs+1 es+1 … ep-1 vp j，这两条路分别对应Dik(k-1) 和Dkj(k-1)。

典型题目：
POJ 2502, POJ 1125, POJ 2240, POJ 1975, POJ 1178, HDOJ 2290

基本模板：

void floyd()
{
for (int k = 0; k < n; ++k)
for (int i = 0; i < n; ++i)
for (int j = 0; j < n; ++j)
if (g[i][k] != inf && g[k][j] != inf)
g[i][j] = min(g[i][j], g[i][k] + g[k][j]);
}
